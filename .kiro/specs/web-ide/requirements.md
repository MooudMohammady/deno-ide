# سند نیازمندی‌ها

## مقدمه

این سند نیازمندی‌های یک IDE تحت وب شبیه VSCode را مشخص می‌کند. این IDE باید قابلیت‌های کامل ویرایش کد، مدیریت فایل‌ها، ترمینال، دیباگ و پلاگین‌ها را داشته باشد و در مرورگر اجرا شود.

## واژه‌نامه

- **IDE تحت وب**: یک محیط توسعه یکپارچه که در مرورگر اجرا می‌شود
- **ویرایشگر کد**: کامپوننت اصلی برای ویرایش فایل‌های کد
- **اکستنشن**: افزونه‌هایی که قابلیت‌های IDE را گسترش می‌دهند
- **ترمینال**: محیط خط فرمان برای اجرای دستورات
- **دیباگر**: ابزار دیباگ برای خطایابی کد
- **اکسپلورر فایل**: نمایش‌دهنده ساختار فایل‌ها و پوشه‌ها

## نیازمندی‌ها

### نیازمندی 1

**داستان کاربر:** به عنوان یک توسعه‌دهنده، می‌خواهم فایل‌های کد را ویرایش کنم، تا بتوانم کد بنویسم و تغییرات ایجاد کنم.

#### معیارهای پذیرش

1. هنگامی که کاربر یک فایل را باز می‌کند، سیستم باید محتوای فایل را در ویرایشگر نمایش دهد
2. هنگامی که کاربر در ویرایشگر تایپ می‌کند، سیستم باید تغییرات را در زمان واقعی نمایش دهد
3. هنگامی که کاربر فایل را ذخیره می‌کند، سیستم باید تغییرات را در فایل اصلی اعمال کند
4. هنگامی که ویرایشگر کد فعال است، سیستم باید هایلایت سینتکس برای زبان‌های برنامه‌نویسی مختلف ارائه دهد
5. هنگامی که کاربر کد را ویرایش می‌کند، سیستم باید تکمیل خودکار کد ارائه دهد

### نیازمندی 2

**داستان کاربر:** به عنوان یک توسعه‌دهنده، می‌خواهم فایل‌ها و پوشه‌ها را مدیریت کنم، تا بتوانم پروژه‌ام را سازماندهی کنم.

#### معیارهای پذیرش

1. هنگامی که کاربر پروژه را باز می‌کند، سیستم باید ساختار فایل‌ها و پوشه‌ها را در اکسپلورر نمایش دهد
2. هنگامی که کاربر روی یک فایل کلیک می‌کند، سیستم باید آن را در ویرایشگر باز کند
3. هنگامی که کاربر یک فایل جدید ایجاد می‌کند، سیستم باید آن را به ساختار پروژه اضافه کند
4. هنگامی که کاربر یک فایل را حذف می‌کند، سیستم باید آن را از ساختار پروژه حذف کند
5. هنگامی که کاربر یک فایل را تغییر نام می‌دهد، سیستم باید نام آن را در ساختار پروژه به‌روزرسانی کند

### نیازمندی 3

**داستان کاربر:** به عنوان یک توسعه‌دهنده، می‌خواهم دستورات ترمینال را اجرا کنم، تا بتوانم پروژه‌ام را بسازم و اجرا کنم.

#### معیارهای پذیرش

1. هنگامی که کاربر ترمینال را باز می‌کند، سیستم باید یک جلسه ترمینال جدید ایجاد کند
2. هنگامی که کاربر دستوری در ترمینال وارد می‌کند، سیستم باید آن را اجرا و خروجی را نمایش دهد
3. هنگامی که کاربر چندین ترمینال باز می‌کند، سیستم باید بین آن‌ها سوئیچ کند
4. هنگامی که ترمینال فعال است، سیستم باید از دستورات پوسته پشتیبانی کند
5. هنگامی که کاربر ترمینال را می‌بندد، سیستم باید جلسه ترمینال را پایان دهد

### نیازمندی 4

**داستان کاربر:** به عنوان یک توسعه‌دهنده، می‌خواهم کد را دیباگ کنم، تا بتوانم باگ‌ها را پیدا و رفع کنم.

#### معیارهای پذیرش

1. هنگامی که کاربر دیباگر را شروع می‌کند، سیستم باید نقاط توقف را تنظیم کند
2. هنگامی که کد در نقطه توقف متوقف می‌شود، سیستم باید مقادیر متغیرها را نمایش دهد
3. هنگامی که کاربر مرحله‌به‌مرحله کد را اجرا می‌کند، سیستم باید اجرا را کنترل کند
4. هنگامی که دیباگر فعال است، سیستم باید کنسول دیباگ را نمایش دهد
5. هنگامی که کاربر دیباگر را متوقف می‌کند، سیستم باید اجرای عادی را از سر بگیرد

### نیازمندی 5

**داستان کاربر:** به عنوان یک توسعه‌دهنده، می‌خواهم اکستنشن‌ها را نصب و مدیریت کنم، تا بتوانم قابلیت‌های IDE را گسترش دهم.

#### معیارهای پذیرش

1. هنگامی که کاربر بازار اکستنشن را باز می‌کند، سیستم باید لیست اکستنشن‌های موجود را نمایش دهد
2. هنگامی که کاربر یک اکستنشن را نصب می‌کند، سیستم باید آن را دانلود و فعال کند
3. هنگامی که کاربر یک اکستنشن را حذف می‌کند، سیستم باید آن را غیرفعال و حذف کند
4. هنگامی که اکستنشن‌ها نصب شده‌اند، سیستم باید قابلیت‌های آن‌ها را در IDE ادغام کند
5. هنگامی که کاربر تنظیمات اکستنشن را تغییر می‌دهد، سیستم باید تغییرات را اعمال کند

### نیازمندی 6

**داستان کاربر:** به عنوان یک توسعه‌دهنده، می‌خواهم UI را شخصی‌سازی کنم، تا بتوانم محیط کاری را به سلیقه خودم تنظیم کنم.

#### معیارهای پذیرش

1. هنگامی که کاربر تم را تغییر می‌دهد، سیستم باید ظاهر IDE را به‌روزرسانی کند
2. هنگامی که کاربر طرح‌بندی را تنظیم می‌کند، سیستم باید موقعیت پنل‌ها را تغییر دهد
3. هنگامی که کاربر کلیدهای میانبر را تنظیم می‌کند، سیستم باید آن‌ها را ذخیره کند
4. هنگامی که کاربر تنظیمات ویرایشگر را تغییر می‌دهد، سیستم باید آن‌ها را اعمال کند
5. هنگامی که UI شخصی‌سازی شده است، سیستم باید تنظیمات را در جلسات بعدی حفظ کند

### نیازمندی 7

**داستان کاربر:** به عنوان یک توسعه‌دهنده، می‌خواهم با Git کار کنم، تا بتوانم نسخه‌های کد را مدیریت کنم.

#### معیارهای پذیرش

1. هنگامی که کاربر مخزن Git را باز می‌کند، سیستم باید وضعیت Git را نمایش دهد
2. هنگامی که کاربر تغییرات را commit می‌کند، سیستم باید commit را ایجاد کند
3. هنگامی که کاربر branch می‌سازد، سیستم باید branch جدید ایجاد کند
4. هنگامی که کاربر merge می‌کند، سیستم باید branch‌ها را ادغام کند
5. هنگامی که کاربر push/pull می‌کند، سیستم باید با remote repository همگام شود

### نیازمندی 8

**داستان کاربر:** به عنوان یک توسعه‌دهنده، می‌خواهم کد را تحلیل کنم، تا بتوانم کیفیت کد را بهبود دهم.

#### معیارهای پذیرش

1. هنگامی که کاربر کد را باز می‌کند، سیستم باید خطاهای سینتکس را هایلایت کند
2. هنگامی که کاربر کد را ذخیره می‌کند، سیستم باید linting را اجرا کند
3. هنگامی که linting اجرا می‌شود، سیستم باید پیشنهادات بهبود را نمایش دهد
4. هنگامی که کاربر کد را فرمت می‌کند، سیستم باید کد را به صورت استاندارد فرمت کند
5. هنگامی که کد تحلیل می‌شود، سیستم باید متریک‌های کد را محاسبه کند
